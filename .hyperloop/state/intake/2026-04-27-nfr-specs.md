## Detailed Findings

### `specs/nfr/api-conventions.spec.md`

Self-identifies as an NFR at line 3: _"NFR: This spec describes REST API conventions that all
bounded context presentation layers MUST follow for consistency."_

Defines URL structure patterns, HTTP status codes, error response format, Pydantic model
conventions, and FastAPI dependency injection rules. These are guidelines that presentation-layer
tasks reference when implementing routes — not a standalone deliverable.

**Decision**: No task. NFR per project policy.

---

### `specs/nfr/architecture.spec.md`

Self-identifies as an NFR at line 3: _"NFR: This spec describes structural constraints enforced
via automated tests, not domain behavior."_

Defines DDD layering rules (domain → ports → application → infrastructure → presentation),
bounded context isolation, shared kernel independence, and composition layer exceptions. These
constraints are enforced via pytest-archon and referenced by implementers during each
bounded-context task.

**Decision**: No task. NFR per project policy.

---

### `specs/nfr/observability.spec.md`

Self-identifies as an NFR at line 3: _"NFR: This spec describes a non-functional architectural
pattern, not domain behavior."_

Defines the Domain-Oriented Observability pattern: probe Protocols, DefaultXxxProbe
implementations with structlog, ObservationContext propagation, probe layering (domain /
application / infrastructure), and structured log configuration (dev vs. production).
Implementers apply these patterns within each bounded context's own tasks.

**Decision**: No task. NFR per project policy.

---

### `specs/nfr/testing.spec.md`

Self-identifies as an NFR at line 3: _"NFR: This spec describes the testing philosophy and
constraints for Kartograph."_

Defines the fakes-over-mocks philosophy, in-memory fake implementation requirements, contract
test expectations, integration test infrastructure rules, acceptable mock targets, and test
layering (domain / application / integration). Implementers apply these conventions when writing
tests for each bounded-context task.

**Decision**: No task. NFR per project policy.
